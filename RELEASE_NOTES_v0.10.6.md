# ByteCase Playbooks v0.10.6 Release Notes

## Content Consistency Sprint

v0.10.6 focuses on credibility and wording quality rather than new feature expansion.

## Updated areas

- Artifact questions
- Chained Coaching paths
- Coach Mode drill questions
- Guardrail wording
- Tool/method language
- README release description

## Artifact guidance

The Artifact section was reviewed for situation/question alignment and more consistent examiner language. The guidance keeps device type and workflow context tighter so unrelated questions are less likely to appear or feel disconnected.

Tool names are kept as common examples of industry tools or methods only. They are not endorsements, recommendations, validation statements, or substitutes for agency policy and local tool validation.

## Coaching paths

Chained Coaching paths were reviewed to strengthen:

- source-first thinking
- scope and authority reminders
- artifact versus actor separation
- cautious BitLocker/RAM language
- mobile source and extraction-limit language
- cloud/source-boundary language
- timestamp/parser disagreement language

The best answer is no longer revealed until the path review, preserving the mentor-style chain.

## Coach Mode drill questions

Built-in drill questions were rewritten for clarity and answer-choice balance. Correct answers are no longer visually signaled by consistently being the longest option.

- Python compile check across Coach question modules
- Python compile check across glossary modules
- Data loading check for playbooks, artifacts, investigative questions, Coach drill questions, Coaching paths, and glossary terms
- Answer-position/answer-length balance check for built-in drill questions and Coaching path questions
- Session export smoke test for JSON, TXT, and DOCX
